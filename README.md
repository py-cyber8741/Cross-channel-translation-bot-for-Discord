# Cross-channel-translation-bot-for-Discord
Cross-channel translation bot for Discord

https://github.com/py-cyber8741/Cross-channel-translation-bot-for-Discord/releases/download/exe/transbot.exe


全133言語に対応したリアルタイム Discord 相互チャンネル翻訳BotおよびGUI管理アプリケーションです。Google翻訳エンジン（deep_translator）による相互翻訳、Google Gemini AI（gemini-2.5-flash）を活用した推敲翻訳機能、スパム検知、トークン暗号化保存機能などを統合しています。
チャンネルAとチャンネルBをグループとし、チャンネルAで発言した内容をチャンネルBに指定した言語に翻訳して出力します。逆も同様です。


1. 初期導入と起動手順
※exe公開の為python等のインストールは不要です。

以下はexe化前の手順。
必要ライブラリのインストール
本アプリケーションの実行には、以下のPythonパッケージが必要です。



Bash
pip install discord.py deep-translator cryptography google-genai

(※GUI機能は標準ライブラリの tkinter を使用します。また、Gemini AI機能は最新の google-genai SDKを使用しています。)
初回起動とセキュリティ仕様
1. スクリプト（*.py）を実行すると、管理GUIが起動します。
2. 初回起動時、トークン暗号化用のキーファイル（secret.key）が自動生成されます。
3. 設定データは multi_server_config.json に保存され、Bot TokenおよびGemini API Keyは自動的に暗号化（Fernet暗号化）されて書き込まれます。
2. GUI管理画面（メインウィンドウ）の操作
階層ツリーとグループ設定
画面中央の「階層ツリー」では、翻訳設定が サーバー（GID） ＞ グループ（Grp） ＞ チャンネル（ChID） の構造で管理されます。
* 追加 / 更新: 上部の入力欄に GID（サーバーID）、Grp（グループ名）、ChID（チャンネルID）、Lan（言語コード）を入力し、「追加/更新」ボタンをクリックします。
* グループ名の変更: 既存のグループを選択して Grp を変更し、「追加/更新」を押すと、グループ名がリネームされます。
* 削除: ツリーで項目を選択して「削除」をクリックすると、指定したチャンネル、グループ、またはサーバー全体の登録を解除します。
* Bot起動: 画面下の「Bot起動」ボタンを押すとバックグラウンドでDiscord Botが接続を開始します。
2つのサイドバーパネル
右上の一覧ボタンから、画面右側にサイドバーを展開できます。


ボタン名称
	機能概要
	操作方法・ヒント
	🌐 言語一覧
	全133言語の正式名称と言語コードを検索・表示
	任意の言語をダブルクリックすると、入力欄の Lan にコードが自動入力されます。
	📡 チャンネル取得
	Botが参加中の全サーバーとテキストチャンネルを一覧表示
	チャンネルをダブルクリックで GID と ChID を自動入力。新着メッセージ数はクリック即時、サーバークリックで3秒後に通知リセットが可能です。
	右クリックメニュー（コンテキストメニュー）
ツリービューのチャンネル上で右クリックすると、以下の操作が可能です。
* 📜 最新ログを表示: 選択したチャンネルの直近メッセージを閲覧・翻訳表示する「ログビューワー」を開きます。
* 💬 このチャンネルに発言: GUI側から直接Discordチャンネルへ発言する「Bot発言ウィンドウ」を開きます。
* 🚪 このサーバーから退出: Botを該当Discordサーバーから退出させます（チャンネル取得サイドバー専用）。
3. 設定ウィンドウ（⚙️ 設定）
メイン画面右上、またはサイドバーの右クリックメニュー「⚙️ 設定」から開きます。
基本設定 (General)
* 🌐 UI 言語: 日本語、英語、中国語（簡体）、韓国語などを切り替えます（未収録の言語を選択した場合、自動翻訳でUIテキストが生成されます）。
* Bot Token: Discord Developer Portalで取得したBotのアクセストークンを入力します（必須）。
* Gemini API Key: Google Geminiを利用した AI翻訳（$ai コマンド）を使用する場合に入力します（任意）。
* ログ取得数: 発言画面（デフォルト5件）およびログビューワー（デフォルト50件）で読み込む履歴数を設定します。
* スパム判定枠: 指定時間（分）内に指定回数以上の発言を行ったユーザーを自動的に検出する基準値を設定します（デフォルト: 1分間に10回）。
BAN設定 / 一時停止リスト
* BAN対象サーバー / ユーザー: IDを追加することで、特定のサーバーやユーザーからの翻訳処理を一切無視・ブロックします。
* 一時停止リスト (Suspended List): スパム検知されたユーザーが自動登録されます。項目を右クリックして「BANリストに登録」するか、「解除」して翻訳を再開させることができます。
4. Discord上でのコマンドと翻訳機能
Discordサーバー内では、管理者およびユーザーが以下のコマンドを使用できます。
相互翻訳グループの対話式設定（管理者専用）


$chset <チャンネル数>

* 使用例: $chset 2 （2つのチャンネル間で相互翻訳グループを作成）
* 動作:
   1. コマンドを実行すると、接続したいチャンネルを選択するドロップダウンメニューが表示されます。
   2. チャンネル選択後、チャット欄に「各チャンネルの翻訳言語コードを60秒以内に入力してください」と指示が出ます。
   3. 例として ja や en を送信すると、自動的にGUIの階層ツリーへ反映され、相互翻訳が開始します。



翻訳グループの削除（管理者専用）

$chdel [チャンネルID]

* 実行したチャンネル（または指定したチャンネルID）が含まれる翻訳グループの登録を削除します。
Gemini AI 意訳・推敲翻訳



$ai <翻訳または推敲させたい文章>

* Google Gemini (gemini-2.5-flash) を呼び出し、発言元チャンネルに設定された言語の自然な文章として翻訳または推敲を行います。
* 生成されたAI文章は、元チャンネルに返信（リプライ）されると同時に、グループ内の他のターゲット言語チャンネルへ自動で翻訳転送されます。
5. 高度なシステム仕様
* メンション・絵文字・URLの自動保護: メッセージ内に含まれるメンション <@id>、チャンネルリンク <#id>、カスタム絵文字 <a:name:id>、URL等は、翻訳エンジンに渡す前に自動的にプレースホルダー（▼0▼ 等）に置換され、翻訳終了後に正確に復元されます。これにより、翻訳によるリンク切れや絵文字の崩れを防ぎます。
* 長文メッセージの自動分割: Discordの文字数制限（2000文字以内／Embed内4000文字以内）を回避するため、通常の翻訳メッセージは 3800文字、AI翻訳の直接返信は 1900文字 を超えた場合に自動で改行位置を判定して分割送信し、ページ番号 (1/2) を付与します。
* ファイル添付の転送: 画像などの添付ファイルが送信された場合、ファイルを自動的に読み取り、翻訳メッセージと共に転送先のチャンネルへ再送信します。

* Bot発言ウィンドウのUndo機能: GUIの「BOT発言ウィンドウ」にある「🗑️ 直近のBot発言を削除」ボタンを押すと、そのチャンネルでBotが直近に発言したメッセージを探索し、即座に削除（取り消し）できます。
